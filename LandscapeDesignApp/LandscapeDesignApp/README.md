# تطبيق تصميم الحدائق (Landscape Design App)

تطبيق أندرويد أصلي (Kotlin + Jetpack Compose) لتصميم الحدائق باستخدام تقنية
الواقع المعزز (ARCore)، بواجهات عربية بالكامل واتجاه RTL.

## هيكل المشروع

| الملف | الوظيفة |
|---|---|
| `ar/ARSessionManager.kt` | إدارة جلسة ARCore، اكتشاف الأرضية، تجميد/استئناف الإطار |
| `ar/AnchorManager.kt` | الوضع المباشر: HitTest ووضع الأنكرات |
| `ar/FrozenFrameProjector.kt` | الوضع المجمّد: إسقاط اللمسة على المستوى المحفوظ (Ray-Plane) |
| `geometry/GeometryUtils.kt` | معادلة Shoelace، حساب المسافات، التقريب |
| `geometry/PolygonClipper.kt` | خصم المساحات (المحددات ومناطق الأشجار) لحساب صافي النجيلة |
| `model/DesignElement.kt` | نماذج البيانات: الأشكال، النباتات، المحددات الثلاثية |
| `model/DesignLayoutState.kt` | الحالة الموحدة لكل خطوات التطبيق |
| `viewmodel/LandscapeViewModel.kt` | منطق الأعمال، التراجع/الإعادة، إعادة الحساب التفاعلي |
| `ui/Step1_AreaCapture.kt` | الخطوة ١: قياس المساحة (مباشر / صورة مجمّدة) |
| `ui/Step2_SoilVolume.kt` | الخطوة ٢: حساب كمية التراب |
| `ui/Step3_DesignStudio.kt` | الخطوة ٣: استوديو التصميم التفاعلي |
| `ui/Step4_LawnCalculation.kt` | الخطوة ٤: مساحة النجيلة الصافية |
| `ui/Step5_ReportScreen.kt` | الخطوة ٥: التقرير النهائي وتصديره |
| `report/ReportGenerator.kt` | مولّد النص السردي التفصيلي باللغة العربية |

## ملاحظات دمج مهمة قبل التشغيل الفعلي

هذا المشروع يقدّم البنية الكاملة والمنطق الرياضي والواجهات بشكل جاهز للإنتاج.
تم تحديث `ui/ArCameraPreview.kt` ليعتمد بالكامل على مكتبة **SceneView
(`ArSceneView`)** بدلاً من الكود اليدوي لـ `GLSurfaceView`/`GLES20`:
عرض خلفية الكاميرا، شبكة اكتشاف الأرضية، وتثبيت/استئناف/إيقاف جلسة ARCore
كلها تتم الآن تلقائياً عبر المكتبة دون أي كود GLES يدوي. `ARSessionManager`
لم يعد يُنشئ الجلسة بنفسه؛ بل يستقبلها عبر `onSessionCreated` من
`ArSceneView`، ويستقبل كل إطار جديد عبر `onSessionUpdated`، بينما تبقى بقية
المنطق (الأنكرات، التجميد، الإسقاط على المستوى المجمّد) كما هي دون تغيير.

نقطتان متبقيتان للتكامل حسب اختيارك:

1. **أيقونة التطبيق** (`@mipmap/ic_launcher`): يجب توليدها عبر Android Studio
   (Image Asset Studio) قبل أول بناء (Build).
2. **استوديو التصميم (الخطوة ٣)**: مقياس البكسل-للمتر في `ScreenMapper` مبسّط
   (بكسل/متر ثابت حول مركز الشاشة) لتوضيح المنطق بشكل قابل للاختبار. في
   الإنتاج الفعلي، يُستبدل بإسقاط حقيقي عبر مصفوفات View/Projection المحفوظة
   من `ARSessionManager`، بنفس أسلوب `FrozenFrameProjector` في الخطوة ١.

جميع المعادلات الرياضية (Shoelace، المحيطات، خصم المساحات، كثافة الزراعة،
تقريب الأعداد لأعلى) مُنفّذة بالكامل ومُختبرة منطقياً وفق المواصفات المطلوبة،
وهي مستقلة تماماً عن طريقة عرض الكاميرا المختارة.
